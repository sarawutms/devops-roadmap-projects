# Dummy Systemd Service

This repository contains the solution for the Dummy Systemd Service project from roadmap.sh. It demonstrates how to create, configure, and manage a custom, long-running systemd service on a Linux system.

🔗 Project URL: [Dummy Systemd Service](https://roadmap.sh/projects/dummy-systemd-service)  

## Project Description

The project consists of a simple Bash script (`dummy.sh`) that simulates a background application by writing a timestamped message to a log file (`dummy-service.log`) every 10 seconds. The script is managed by a systemd unit file (`dummy.service`), which ensures the service starts on boot, runs in the background, and automatically restarts if it encounters any failures.

## Files Structure

- `dummy.sh`: The main bash script that runs in an infinite loop.
- `dummy.service`: The systemd configuration file.
- `dummy-service.log`: The log file generated by the script (ignored via `.gitignore`).

## Prerequisites

- A Linux distribution with `systemd` installed (e.g., Ubuntu).
- Sudo/root privileges to manage system services.

## Installation & Setup

**1. Clone the repository**
```bash
git clone [https://github.com/sarawutms/devops-roadmap-projects.git](https://github.com/sarawutms/devops-roadmap-projects.git)
cd devops-roadmap-projects/dummy-systemd-service
```
**2. Make the script executable**
```bash
chmod +x dummy.sh
```
**3. Link or copy the service file to the systemd directory**
```bash
sudo cp dummy.service /etc/systemd/system/
```
**4. Reload the systemd daemon to recognize the new service**
```bash
sudo systemctl daemon-reload
```
**5. Start and enable the service**
```bash
sudo systemctl start dummy
sudo systemctl enable dummy
```
## Usage & Management
Use the following commands to interact with the service:  
- Check Service Status:
```bash
sudo systemctl status dummy
```
- Stop the Service:
```bash
sudo systemctl stop dummy
```
- View Systemd Logs (Real-time):
```bash
sudo journalctl -u dummy -f
```
- View Custom Application Logs:
```bash
tail -f dummy-service.log
```
## Cleanup
To completely remove the service from the system:
```bash
sudo systemctl stop dummy
sudo systemctl disable dummy
sudo rm /etc/systemd/system/dummy.service
sudo systemctl daemon-reload
```
