# Dummy Systemd Service

This repository contains the solution for the [Dummy Systemd Service project](https://roadmap.sh/projects/dummy-systemd-service) from roadmap.sh. It demonstrates how to create, configure, and manage a custom, long-running systemd service on a Linux system.

## Project Description

The project consists of a simple Bash script (`dummy.sh`) that simulates a background application by writing a timestamped message to a log file (`dummy-service.log`) every 10 seconds. The script is managed by a systemd unit file (`dummy.service`), which ensures the service starts on boot, runs in the background, and automatically restarts if it encounters any failures.

## Files Structure

- `dummy.sh`: The main bash script that runs in an infinite loop.
- `dummy.service`: The systemd configuration file.
- `dummy-service.log`: The log file generated by the script (ignored via `.gitignore`).

## Prerequisites

- A Linux distribution with `systemd` installed (e.g., Ubuntu).
- Sudo/root privileges to manage system services.

## Installation & Setup

**1. Clone the repository**
```bash
git clone [https://github.com/sarawutms/devops-roadmap-projects.git](https://github.com/sarawutms/devops-roadmap-projects.git)
cd devops-roadmap-projects/dummy-systemd-service
```
**2. Make the script executable**
```bash
chmod +x dummy.sh
```
**3. Link or copy the service file to the systemd directory**
```bash
sudo cp dummy.service /etc/systemd/system/
```
